# Security Policy

## Supported Versions

This project is currently pre-1.0. Security fixes are applied to the latest commit on `main` unless release branches are introduced later.

## Reporting A Vulnerability

Please report security issues by opening a GitHub issue in this repository using the bug report template.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Relevant logs, screenshots, or request examples.

Do not include real secrets, private account codes, or sensitive production data in reports.

## Account Codes

The app intentionally avoids registration. Account codes are bearer credentials: anyone with an account code can access that account's loadouts. Treat account codes like passwords.
