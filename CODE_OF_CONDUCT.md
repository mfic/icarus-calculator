# Code of Conduct

## Our Pledge

We as contributors, maintainers, and community members pledge to make participation in this project a respectful, harassment-free experience for everyone.

We welcome people regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, sexual identity and orientation, socioeconomic status, or technology choices.

## Expected Behavior

Examples of behavior that contributes to a positive environment:

- Use welcoming and inclusive language.
- Be respectful of differing viewpoints and experiences.
- Give and accept constructive feedback gracefully.
- Focus criticism on ideas, code, documentation, and behavior, not people.
- Assume good intent, but take responsibility for impact.
- Keep discussions relevant to the project and its users.

## Unacceptable Behavior

Examples of unacceptable behavior:

- Harassment, intimidation, threats, stalking, or sustained disruption.
- Insults, personal attacks, slurs, or derogatory comments.
- Sexualized language or imagery, unwanted sexual attention, or inappropriate advances.
- Publishing private information without explicit permission.
- Trolling, baiting, or deliberately derailing discussions.
- Repeatedly ignoring maintainer requests to stop harmful behavior.

## Scope

This Code of Conduct applies in all project spaces, including GitHub issues, pull requests, discussions, documentation, code review, release work, and any project-related communication.

It also applies when someone is representing the project publicly, such as by using an official project account, presenting project work, or acting as a maintainer.

## Reporting

Report Code of Conduct concerns by opening a GitHub issue in this repository using the bug report template. Include as much context as you can, such as links, screenshots, usernames, dates, and a short description of the impact.

Maintainers should handle reports promptly, respectfully, and with care. Reports that are abusive, intentionally false, or made in bad faith may themselves be treated as a conduct issue.

## Enforcement

Maintainers are responsible for clarifying standards of acceptable behavior and may take fair, proportionate action in response to unacceptable behavior.

Possible enforcement actions include:

- A private or public clarification.
- A warning.
- Editing or removing comments, commits, documentation, issues, or pull requests.
- Temporary restriction from project interaction.
- Permanent restriction from project interaction.

Enforcement decisions should consider severity, impact, prior behavior, and the safety of the project community.

## Attribution
